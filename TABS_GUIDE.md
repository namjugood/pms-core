# 탭 기능 가이드

## 탭 시스템 개요

여러 개의 탭을 만들어 각각 독립적인 데이터를 관리할 수 있습니다. 예를 들어:
- 프로젝트별로 탭 생성
- 팀별로 탭 분리
- 분기별로 탭 구분

## 탭 관리

### 1. 새 탭 만들기

**방법:**
- 탭 네비게이션 바 오른쪽의 `+` 버튼 클릭

**결과:**
- 새 탭이 "탭 N" 이름으로 생성됨
- 자동으로 새 탭으로 전환됨
- 빈 테이블로 시작

### 2. 탭 이름 변경

**방법 1: 더블클릭**
1. 변경하려는 탭 이름을 더블클릭
2. 프롬프트 창에 새 이름 입력
3. 확인 클릭

**팁:**
- 프로젝트명, 팀명, 날짜 등으로 구분하면 편리합니다
- 예: "2024 Q1 백엔드팀", "프로젝트 A", "개발팀 평가"

### 3. 탭 전환

**방법:**
- 전환하려는 탭을 클릭

**특징:**
- 각 탭은 독립적인 테이블 데이터를 가집니다
- 탭을 전환해도 데이터는 유지됩니다

### 4. 탭 닫기

**방법:**
- 탭 우측의 `×` 버튼 클릭
- 확인 메시지에서 확인 클릭

**주의:**
- 최소 1개의 탭은 유지됩니다
- 닫은 탭의 데이터는 **저장하지 않으면 사라집니다**
- 저장 후 닫는 것을 권장합니다

## 저장 및 불러오기

### 모든 탭 저장

**File → Save (Ctrl+S)**
- 모든 탭의 이름과 데이터가 함께 저장됩니다
- 탭 순서도 유지됩니다

**저장 데이터 구조:**
```
파일.dat
├── 탭 1 (이름 + 테이블 데이터)
├── 탭 2 (이름 + 테이블 데이터)
├── 탭 3 (이름 + 테이블 데이터)
└── ...
```

### 모든 탭 복원

**File → Load (Ctrl+O)**
- 저장된 모든 탭이 그대로 복원됩니다
- 탭 이름, 순서, 데이터 모두 유지
- 기존 탭은 모두 대체됩니다

### 구 버전 파일 호환

- 탭 기능 이전에 저장한 파일도 불러올 수 있습니다
- 구 버전 파일은 자동으로 "시스템개발" 탭으로 변환됩니다

## 사용 시나리오

### 시나리오 1: 프로젝트별 관리

```
[프로젝트 A] [프로젝트 B] [프로젝트 C] [+]
```

각 프로젝트의 기여도를 별도 탭으로 관리

### 시나리오 2: 팀별 관리

```
[백엔드팀] [프론트엔드팀] [인프라팀] [+]
```

팀별로 평가 항목을 분리

### 시나리오 3: 시간별 관리

```
[2024 Q1] [2024 Q2] [2024 Q3] [2024 Q4] [+]
```

분기별로 평가 데이터 구분

## 팁 & 트릭

### 효율적인 탭 사용법

1. **명확한 이름 사용**
   - ❌ "탭 1", "탭 2"
   - ✅ "백엔드개발", "프론트개발"

2. **정기적인 저장**
   - 탭을 추가하거나 수정 후 `Ctrl + S`로 저장
   - 탭을 닫기 전에 반드시 저장

3. **탭 순서 활용**
   - 중요한 탭을 앞쪽에 배치
   - 현재 작업 중인 탭을 왼쪽에

4. **불필요한 탭 정리**
   - 사용하지 않는 탭은 닫아서 깔끔하게 유지
   - 단, 닫기 전 저장 확인!

### 단축키 활용

- `Ctrl + S`: 현재 모든 탭 저장
- `Ctrl + O`: 파일 불러오기
- 탭 이름 더블클릭: 빠른 이름 변경

## 주의사항

⚠️ **데이터 손실 방지**
- 탭을 닫기 전 반드시 저장
- 저장하지 않고 닫으면 데이터 복구 불가능
- 작업 중 자주 `Ctrl + S` 습관화

⚠️ **최소 탭 개수**
- 최소 1개의 탭은 항상 유지됩니다
- 마지막 탭은 닫을 수 없습니다

⚠️ **파일 호환성**
- 새 버전에서 저장한 파일은 구 버전에서 열 수 없습니다
- 구 버전 파일은 새 버전에서 정상 작동합니다
