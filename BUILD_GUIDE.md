# 빌드 및 배포 가이드

## 📦 EXE 파일 생성 방법

### 1단계: 빌드 준비

프로젝트가 정상적으로 작동하는지 확인:

```powershell
npm run dev
````

### 2단계: EXE 파일 생성

```powershell
npm run dist
```

이 명령은 다음 작업을 수행합니다:

1.  TypeScript 컴파일 (`node build.js`)
      - `src/renderer` 내의 모든 리소스(js, css, html)를 `dist`로 복사
2.  Windows 설치 파일 생성 (`electron-builder`)

### 3단계: 생성된 파일 확인

빌드가 완료되면 `release` 폴더에서 다음 파일을 찾을 수 있습니다:

```
release/
├── 기여도 평가 점수 프로그램 Setup 2.1.0.exe  (설치 파일)
└── win-unpacked/                               (압축 해제된 버전)
    └── 기여도 평가 점수 프로그램.exe
```

## 📝 배포 전 체크리스트

새로운 버전(v2.1.0) 배포 전 다음 사항을 반드시 확인하세요.

  - [ ] **기본 기능**

      - [ ] 프로그램 실행 및 종료 정상 작동
      - [ ] 탭 추가/삭제/이름 변경
      - [ ] 데이터 입력 시 실시간 계산 반영

  - [ ] **PDF 내보내기 (신규)**

      - [ ] `Ctrl+P` 또는 메뉴로 미리보기 창이 뜨는지 확인
      - [ ] 데이터가 문서 양식(표)에 맞게 표시되는지 확인
      - [ ] 기간(최소\~최대) 및 출력일이 헤더에 정확히 찍히는지 확인
      - [ ] `Ctrl+휠`로 미리보기 확대/축소가 부드럽게 되는지 확인
      - [ ] 실제 PDF 파일로 저장되고, 내용이 깨지지 않는지 확인

  - [ ] **사용자 가이드 (신규)**

      - [ ] `F1` 키로 가이드 팝업이 뜨는지 확인
      - [ ] 상단 탭(가이드/업데이트 내역) 전환이 잘 되는지 확인
      - [ ] 스타일이 깨지지 않고 다크 모드로 잘 나오는지 확인

  - [ ] **설정 확인**

      - [ ] `package.json` 버전이 `2.1.0`인지 확인

## 🔧 빌드 문제 해결

### 문제 1: PDF 미리보기 스타일이 이상함

**원인:** `pdf.css` 파일이 `dist` 폴더로 복사되지 않았을 수 있습니다.
**해결:** `build.js` 실행 로그를 확인하거나 `dist/renderer/styles/pdf.css` 파일이 존재하는지 확인하세요.

### 문제 2: "build/icon.ico not found"

**해결:** 아이콘 파일을 `build/icon.ico`에 배치하거나 `package.json`에서 아이콘 설정을 제거하세요.

### 문제 3: dist 폴더가 없음

**해결:** `node build.js`를 먼저 실행하여 `dist` 폴더를 생성하세요.

## 📦 배포 방법

### 방법 1: 직접 배포 (권장)

1.  `release/기여도 평가 점수 프로그램 2.1.0.exe` 파일을 배포
2.  사용자가 다운로드 후 더블클릭으로 바로 실행 (설치 불필요)

### 방법 2: 공유 폴더/클라우드

  - 사내 공유 폴더나 클라우드 스토리지에 업로드하여 링크 공유
